About

RETAIL.BASE is a modular and extensible framework designed to facilitate rapid development of enterprise-grade applications. It provides a set of reusable components, libraries, and tools that streamline common development tasks and promote best practices.

Features

  • Layered architecture: API, business, data, and shared models
  • Modular projects: API, NEG (business), DAT (data), OBJ (models)
  • Reusable components and libraries for common development tasks
  • Modern ASP.NET Core Web API with JWT authentication
  • API versioning (v1, v2) and Swagger/OpenAPI
  • Repository pattern and EF Core for SQL Server
  • Comprehensive unit tests (xUnit, Moq)
  • CI/CD pipeline (Jenkins)
  • Extensible for new modules and business logic
  • Promotes best practices for enterprise-grade solutions

Tech Stack

Backend

.NET 8, ASP.NET Core, Entity Framework Core

Database

SQL Server (default), migrations enabled

Testing

xUnit, Moq, coverlet

DevOps

Jenkins, systemd, rsync

Usage

  • Build: dotnet build RETAIL.BASE.sln
  • Run API: dotnet run --project RETAIL.BASE.API/RETAIL.BASE.API.csproj
  • Swagger UI: Enabled by default for API exploration
  • Configure: Edit RETAIL.BASE.API/appsettings.json for connection strings and JWT keys
  • Test: dotnet test RETAIL.BASE.NEG.Tests/RETAIL.BASE.NEG.Tests.csproj
  • CI/CD: Jenkins pipeline for build, publish, deploy, and restart

Getting Started

Clone the Repository

Clone the project from GitHub:

git clone https://github.com/mrodriguex/RETAIL.BASE.git
cd RETAIL.BASE

Prerequisites

  • .NET SDK 8.0 (for building and running the solution)
  • .NET SDK 10.0 (for running tests)
  • SQL Server instance (for database)
  • Node.js and npm (for the web frontend)

Compile and Build

Build the entire solution:

dotnet build RETAIL.BASE.sln

Or build the API only:

dotnet build RETAIL.BASE.API/RETAIL.BASE.API.csproj

Debug Locally

Run the API locally:

dotnet run --project RETAIL.BASE.API/RETAIL.BASE.API.csproj

Access Swagger UI at http://localhost:5000/swagger (or configured port).

For the web frontend:

cd RETAIL.BASE.WEB
npm install
npm run dev

Access the web app at http://localhost:5173 (Vite default).

Deploy

The project includes a Jenkins CI/CD pipeline for automated deployment:

  • Builds and publishes the API for Linux
  • Deploys via SSH and rsync
  • Restarts the systemd service

See Jenkinsfile for details. For manual deployment, publish and deploy to your server.

API Overview

Key endpoints (v1, v2):

  • Auth (v1/v2): POST /api/v1/auth/login, POST /api/v2/auth/login
  • User (v1/v2): GET /api/v1/user, POST /api/v1/user, PUT /api/v1/user/{id}, DELETE /api/v1/user/{id}
  • Customer (v1): GET /api/v1/customer, POST /api/v1/customer, PUT /api/v1/customer/{id}, DELETE /api/v1/customer/{id}
  • Company (v1): GET /api/v1/company, POST /api/v1/company, PUT /api/v1/company/{id}, DELETE /api/v1/company/{id}
  • Role (v1): GET /api/v1/role, POST /api/v1/role, PUT /api/v1/role/{id}, DELETE /api/v1/role/{id}
  • MenuItem (v1): GET /api/v1/menuitem
  • Config (v1): GET /api/v1/config
  • Cryptographer (v1): POST /api/v1/cryptographer/encrypt, POST /api/v1/cryptographer/decrypt
  • Reports (v1): GET /api/v1/rpt/... (various report endpoints)

See README for request/response examples and full API documentation.
